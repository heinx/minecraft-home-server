# AGENTS.md project description

Minecraft Bedrock server which is easy to install and runs reliably on a home Linux Ubuntu server

Github repo: https://github.com/heinx/minecraft-home-server

## Overview

The task is to create a generalized version of this setup that can be pushhed to vsc and deployed to a server via git. The new version should be more modular and configurable, with less hardcoded paths and values.

## Key requirements

### The basic functions that must be preserved are:

- Hooking to the systems service manager and automatically running the minecraft server at all times
- Backing up the world and server properties on a nightly schedule
  - The backup should be also copied to a third party location, e.g. Google Drive, to protect against system failure
- Automatically checking for updates and applying them on a nightly schedule
  - It seems that the update script has broken because Microsoft has changed the download page to not immediately contain the download link in the HTML, but instead it is possibly generated by JS. This should be fixed.
- Logs should be preserved across updates and easily accessible

### New functionality that should be added:

- Configuration of the server: Name, world name, user, paths, backup location, update schedule etc.
- Notifications via email if update fails, backup fails, or server goes down
- The system should be possible to initialise with an existing world and server properties.

## Package delivery

The new version packages should be released in Github and include authentication against the Github repo using e.g. gh attestations so that users of the software can verify the authenticity of the package. The package should be installable via a single command, e.g. `curl -sL https://github.com/username/repo/releases/latest/download/package.sh | bash`.

Gitgub actions should package the relevant files into an artifact

## Verification

The system should be tested on a virtual machine running a common Linux, e.g. Ubuntu. Tests need to be easily repeatable to have a short feedback loop. The tests and VM build and teardown need to be automated.


The tests need to cover:

- The server starts correctly and is accessible
- The service manager hooks are working and the server restarts if it goes down
- Backups are created correctly and can be restored
- Updates are applied correctly and the server continues to run after update

## sub-agents

Create a plan of what kind of agents to use. For example one can design the service installation and test setup, one can design and implement the update and backups.
